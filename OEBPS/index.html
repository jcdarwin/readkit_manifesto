Readk.it: digital reading simplified

Readk.it is a responsive EPUB reading system that takes advantage of the abilities of modern web browsers to allow an attractive reading experience on all manner of devices

  • The Essay

Readk.it is a light-weight and open-source browser-based reading system that can be combined with EPUB content to produce a book-like experience in a single HTML file. Readk.it can also be used to serve a library of EPUB titles from a website under the publisher’s control.

Because Readk.it is browser-based, it allows publishers to use modern web design techniques in eBook production. This means the publishers can have more control over the distribution and appearance of their publications across smartphones, tablets and desktop computers, while still making these same publications available conventionally via existing vendor EPUB reading systems.

What’s wrong with eBooks?

Well, where to start? Ebooks are a great medium for both publishers and readers, allowing texts to be viewed across a wide range of different screen sizes, and allowing publishers to sell discrete bundles of content to a willing market.

However, although a number of channels exist through which publishers can sell their eBook content, the capabilities of most vendor eBook reading systems are still below par, forcing publishers to compromise on quality and market reach.

Specific problems that publishers — and by extension readers — face include:

  • reading systems that aren’t able to keep up with recent developments in displaying rich and engaging digital content
  • content that looks and performs differently from one vendor reading system to another, due to incompatibilities and different implementations of features
  • reading systems that lock both publishers and readers to a particular vendor and their devices, reducing market reach
  • eBook standards (EPUB2 & EPUB3) that are full of complexity, and difficult to implement in a standard fashion across different vendor reading systems

The pace of change

Modern eBook publishing is built on web standards, and these standards are continually evolving. The current versions of the web standards lynchpins — HTML5 and CSS3 — are still not complete, and even when they are will represent only milestones on a constantly-evolving road.

The EPUB standard is based on these web technologies, however the investment of resources and effort required to keep reading systems current with the ongoing development of web standards is beyond the means or interests of many vendors. The result is a range of reading systems that only allow a fraction of the functionality now available in modern browsers.

With advances in areas such as typography, animation and metadata, web browser development continues apace and is backed by large communities of effort, both privately funded and open sourced. With recent developments such as WebRTC, the possibilities for technologies such as real-time communication, speech recognition, and gesture-based interaction are opening up, and these offer many opportunities for publishers producing digital content.

Modern browser support

The browser is the platform

Modern browser support

Web browsers that support modern web standards make so much possible. With web standards continually emerging and improving, browsers are — more than ever — becoming the platform. Much of the design and presentation that is essential to quality eBook production can be acheived today with browsers such as Webkit (Safari and Chrome), Firefox, and Internet Explorer 10+.

Market reach and vendor lock-in

Publishers want to make their content available to a wide audience, while readers want easy access to a wide range of this content. Given that these motives are aligned, it’s disappointing that many of the available vendor reading systems effectively lock publishers and readers into a single channel. Publishers can develop alternate versions of their content for other channels, and readers can use multiple reading systems and devices to access content provided through different channels, however both would rather have a single version of content that works across the widest possible range of devices.

With most computing devices these days from smartphones and tablets through to desktop computers featuring a web browser, there already exists something resembling a single platform accessible by the largest part of the market. The range of modern web browsers commonly in use (Firefox, Chrome, Safari, and Internet Explorer 10+) is sufficiently narrow for content developers to be able to produce a single version of content that will work well across all of these, and which will therefore be viewable by readers across a wide range of devices and screen sizes.

Although there will always be differences between these browsers, web designers these days have an easier job of producing content that works consistently across the various browsers and devices than do eBook designers trying to produce EPUB content that works consistently as well across the different vendor reading systems.

EPUB3: a complex standard

The EPUB3 standard provides a rich specification for packaging eBook content, but one that is also perhaps overly complex in a number of areas. Reading system vendors are finding it hard to offer full and consistent support for the functionality allowed by EPUB3, meaning that it is unlikely that complex content will appear seamlessly and similarly across the different vendor reading systems.

Given that EPUB3 is based on HTML5 and CSS3 and allows the use of scripting technologies, we can turn to the modern web browser as a solution for rendering the content included in an EPUB3 eBook. The missing piece of the puzzle however, is a means of providing structured access and navigation to this content; although it is easy to uncompress an EPUB3 file and open the constituent pages seperately in a web browser, the browser does not inherently know about the navigation and content structure employed by the EPUB3 content, and so is not able to provide seamless reading of the entire publication without some help.

Readk.it: a solution

Using modern web design techniques, coupled with Readk.it as a browser-based reading system, we can produce eBooks that provide a book-like experience in the web-browser, but which will also fallback to working conventionally in vendor-supplied reading systems.

Importantly, browser-based display of the eBook content provides for a reading experience which is both richer than can be achieved in vendor reading systems, and is also accessible by a wide audience in a consistent fashion.

This solution relies on the following principles:

  • The use of modern web design techniques to allow the content to intelligently adapt to the shape of the container, whether this be a browser on a smartphone, tablet or desktop computer, or whether it is a vendor’s reading system application.
  • Making use of the standards-based support of modern web browsers to provide rich typography, embedded multimedia, and interactivity and animation
  • The use of a reading system embedded inside the eBook, and which can be used by the browser to display the content according to the publisher’s wishes.

Responsive web design

Responsive web design involves the use of CSS media queries to allow content to behave differently based on the size of the browser screen.

With CSS media queries having across-the-board support from all major browsers, as well as from many vendor reading systems, it is possible to design one version of eBook content that will adapt gracefully to the size of the screen on which it is viewed.

Responsive web design

Books that look beautiful on devices of every size

Responsive web design

Responsive web design techniques allow us to create books that take advantage of as much or as little screen real estate as is available on the user’s device.

Through the use of media queries, it’s possible to create digital publications that adapt gracefully to the size of the user’s device.

Media queries are implemented using CSS, and help to ensure optimal readability on any screen size.

Rich content

Web standards have evolved to the point where they support rich typography, native support for video and audio, and animation and interactivity that supports the content. Modern web browsers now provide reasonably consistent implementations of these standards, and the inclusion of these web browsers on many computing devices means that publishers can produce rich content that provides a similar experience for a wide audience.

Animation and interactivity

Promoting engagement with the content without being obtrusive

Animation and interactivity

Animaton and interactivty, when used appropriately, allow the user to intuitively interact with the content. Provided by both CSS and JavaScript, animation and interactivity can help provide a seamless and attractive content navigation experience.

Multimedia, in the form of video and audio, can illustrate and elucidate the content, providing visual and audible examples that reinforce and support the text.

An embeddable reading system

As EPUB content is created using common web standards (HTML, CSS and JavaScript), a reading system can be developed using the same technologies, and embedded inside an EPUB file. This reading system can then be used by a web browser to allow structured navigation through the content.

There already exist a small number of projects such as the IDPF’s Readium and monocle which target the browser as a reading platform, however Readk.it tries to improve on these in a number of ways by:

  • having the reading system able to travel with the content inside a single file, whether this be an EPUB or an HTML file, and so preserving the ability for the content to be presented as the publisher intended
  • emphasising the role of responsive design in allowing an attractive reading experience that works well across a wide range of screen sizes.
  • eschewing those skeuomorphisms which are only relevant in the printed world, such as page turn
