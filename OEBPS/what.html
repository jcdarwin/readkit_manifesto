What is Readk.it?

Readk.it is:

  • a reading system which can serve a library of EPUB titles
  • a reading system which can be combined with the content into a single EPUB or HTML file and travel with it
  • lightweight and easily-customisable by the publisher to take advantage of browser capabilities
  • a reading system which tries to move away from some of the skeuomorphisms of current vendor reading systems, instead taking advantage of the possibilities of digital reading.

An embeddable reading system

Having the reading system travel with the eBook means that publisher’s intentions regarding the presentation of their content will always be honoured when a web browser is used to read the eBook.

At the same time, as the EPUB file remains fully compliant with the EPUB standard, it is able to be read by any vendor reading system that recognises EPUB content (although, of course, possibly with less richness of content than is available when using Readk.it itself to view the content).

Although excelling as a reading system that can be customised and embedded in a particular eBook publication, Readk.it can also be easily configured to web-serve a library of titles.

A lightweight and easily-customisable reading system

As Readk.it is built using open web technologies (HTML, CSS and JavaScript), it is easy for a competent web developer to alter the behaviour of Readk.it as they desire. This may include:

  • changing the Readk.it chrome, such that some features are disabled, or other features added.
  • customising the branding and user interface of Readk.it to reflect their preferences.
  • customising Readk.it to support special features of the content.

Readk.it uses only client-side JavaScript to perform its work; as far as a web server is concerned, all of the Readk.it files and the EPUB/s that Readk.it is presenting are static. This means there is no requirement for server-side logic, and all of Readk.it and the EPUB files that it is presenting can be served from cache, or solely from a cloud-based storage service such as Amazon’s S3.

Readk.it does use local storage and app cache to store user preferences and EPUB data client-side, in order to allow the user to read their content in multiple sessions. The first time a publication is accessed using Readk.it, the entire EPUB file is loaded into app cache storage, thus allowing the user to continue reading the publication in future while offline.

This approach does mean that Readk.it is limited by the size of the publication; if your content requires more than a reasonable amount of multimedia (particularly video), you might like to think about dividing your content up into separate publications.

A reading system for digital reading

Readk.it avoids some of the more obvious ways that features necessary in printed books have been re-implemented in other digital reading systems, but without adding obvious value. Like the Landau roofs used on early motorcars, many of these features, such as page-turn and faux wooden bookshelves tend towards the decorative rather than functional.

In contrast to page-based presentation of many current vendor reading systems, Readk.it by default presents a chapter as a single folio, allowing the reader to read the chapter content by continuously scrolling down the content, but also to access other chapters by scrolling across.

When the reader swipes back to a chapter they have previously been reading, they are positioned at the same place as they were at previously.
