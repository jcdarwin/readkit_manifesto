What does Readk.it allow?

Readk.it has been created in order to:

  • provide publishers with a freely-available and easy means of serving their readers with truly responsive and attractive eBook content from their own website, on their own terms
  • allow publishers to have more control over consistent presentation of their content.
  • allow publishers to easily provide a customised reading experience for a given title.
  • provide publishers with a means of embedding a reading system inside an EPUB or HTML file, and thereby letting it travel with the eBook. By doing this, publishers can ensure that their content is displayed faithfully when viewed in a modern web browser, but also remain valid and work conventionallly in existing vendor EPUB reading systems
  • allow publishers to easily tap into advances in web standards and browser technology
  • provide a reading system which is tailored for use on mobile devices, but also works well in desktop browsers.
  • allow publishers to reach a wider audience using a single content file

Consistent presentation of content

Although there exist different makes of web browsers, browser manufacturers (including Microsoft with Internet Explorer 10+) are doing a good job of standardising on the implementation of web technologies, such that consistent presentation of content is typically easier to acheive than it is with vendor reading systems.

Using technologies such as CSS media queries, it is possible to produce a single EPUB or HTML file of content which will work well across a range of different device sizes, from smartphones to desktop displays.

A customised reading experience

As Readk.it travels within the EPUB or HTML file, the reading experience can be customised as necessary for each title.

Through the use of media queries, different features can be presented for a given screen size; for instance, content can be presented more simply on a smartphone screen than on a tablet or desktop display, with certain content elements being hidden or presented in a simplified fashion.

Tap into advances in browser technology

As Readk.it is browser-based, when web browsers adopt newer features, these are immediately available for use by the content publisher.

Contrast this with the many vendor reading systems that provide only slow and fitful uptake of technology advances.

Typography that is both beautiful and functional

Web fonts: beautiful typography

Web fonts allow us to apply traditional typographic practices to eBook publishing. With an ever-expanding range of fonts available for embedding in eBooks, along with with improvements in CSS and innovations using JavaScript, digital typography is coming of age.

Web-based typography can be truly striking and innovative, and although there are still areas needing further development, others are coming along nicely. In fact, with the CSS3 Paged Media Module, we’ll soon be experiencing web technologies being used for typsetting books intended for print.

Searchable metadata

EPUBs can contain significant amounts of rich metadata about the publication. Conventionally, this metadata is locked up in the EPUB file. However when web-serving publications, Readk.it can retrieve this metadata and embed it in the web-served publication such that is discoverable by search engines.

What’s more, with the advent of metadata standards such as schema.org’s use of the microdata-based Book schema, metadata can be presented to search engines that are specifically looking for information in the context of a book.

Providing discoverability in a sea of content

Metadata

Appropriate metadata helps to ensure that the reader can find the content, and once having found it, navigate easily and appropriately through the content.

Discoverability of content is becoming one of the crucial requirements of online publishing, and first and foremost this means ensuring that search engines such as Google know how to find your content.
